## Batch 1: Pure Computational Extensions (~530 lines)

### Feature 1: POTH (Precision of Treatment Hierarchy)
- **What**: Single metric (0-1) quantifying ranking certainty. POTH=1 means perfect certainty; POTH=0 means total ambiguity.
- **Algorithm**: Assemble rank probability matrix P from `AppState.ranking` (each element has `.rankProbs` array):
  ```
  P = AppState.ranking.map(r => r.rankProbs)  // T x T matrix
  // T = number of treatments = number of possible ranks
  H_treatment_i = -sum_j(P[i][j] * log2(P[i][j]))  for each treatment i
  H_observed = (1/T) * sum_i(H_treatment_i)
  H_max = log2(T)
  POTH = 1 - H_observed / H_max
  ```
  Handle p=0 with convention 0*log2(0)=0 (use `p > 0 ? p * Math.log2(p) : 0`).
- **Location**: New class `POTHCalculator`. Output injected into existing ranking results as a stat card alongside P-scores/SUCRA. Also reported in ranking summary table.
- **Dependencies**: `AppState.ranking[i].rankProbs` — no new data input needed.
- **Estimated lines**: ~80
- **Citation**: Chiocchia et al. 2025, Statistics in Medicine.

### Feature 2: MID-Adjusted Rankings
- **What**: Rankings accounting for minimally important differences. Treatments within MID of each other are tied.
- **Algorithm**:
  1. User provides MID threshold (default 0.5 for log-OR; configurable via input field)
  2. In each Monte Carlo simulation (reuses existing rank simulation infrastructure):
     - For pair (A,B): if |effect_A - effect_B| < MID, assign tied rank (average rank)
  3. Recompute P-scores from tied-rank simulations → "MID-P-score"
  4. Compute "P(meaningfully best)" = fraction of sims where treatment has rank 1 AND is MID-better than rank 2
  5. Tabulate: original P-score, MID-P-score, P(meaningfully best), mean MID-rank
- **Location**: New class `MIDAdjustedRanking`. New collapsible section in Rankings tab titled "MID-Adjusted Rankings" with:
  - Input field for MID value (with tooltip explaining defaults by effect measure)
  - Toggle button to run MID-adjusted analysis
  - Comparison table: original vs MID-adjusted rankings side-by-side
- **Dependencies**: Existing rank simulation from `FrequentistNMA`/`BayesianNMA`. No dependency on POTH (separate display areas).
- **Estimated lines**: ~200
- **Citation**: Spineli et al. 2025 (mid.nma.rank package concept).

### Feature 3: Contribution Matrix (Upgrade)
- **What**: Upgrade existing `ContributionMatrix` (line ~2724) from heuristic weight-based approach to proper hat-matrix method (Papakonstantinou 2018). Existing `ContributionMatrix` object, `renderContributionMatrix`, and `contributionMatrixContainer` are replaced in-place.
- **Algorithm**: GLS hat matrix approach (Rücker 2012, Papakonstantinou 2018):
  ```
  H = X (X' Sigma^{-1} X)^{-1} X' Sigma^{-1}
  ```
  where X = design matrix, Sigma = full variance-covariance matrix (including off-diagonal terms for multi-arm studies). Use `fitGLS().Si` (Sigma-inverse) which is already computed in the NMA engine and correctly handles multi-arm correlations.
  - Each row of H corresponds to a study; columns correspond to basic contrasts
  - Aggregate rows by comparison (edge) to get edge-level contributions
  - Express as percentages (rows sum to 100% for each network estimate)
  - Note: Using diagonal-only W would be incorrect for multi-arm studies; Sigma^{-1} from GLS is required.
- **Location**: Replaces existing `ContributionMatrix` class in-place. Existing UI container `contributionMatrixContainer` and renderer `renderContributionMatrix` are upgraded.
  - Enhanced heatmap table: rows = network comparisons, columns = direct comparisons
  - Color intensity = contribution percentage (white→blue gradient)
  - Downloadable as CSV
- **Dependencies**: Design matrix X and Sigma^{-1} from `fitGLS()` in `FrequentistNMA`. Only available for frequentist models.
- **Estimated lines**: ~250 (net change ~150 after replacing existing)
- **Citation**: Papakonstantinou et al. 2018; Rücker 2012.

## Batch 2: New Visualizations & Framework Extensions (~650 lines)

### Feature 4: Hasse Diagrams (Partial Treatment Order)
- **What**: DAG showing which treatments are definitively ranked above others vs. incomparable.
- **Algorithm**:
  1. For each treatment pair (A, B): test if 95% CI of d_AB excludes 0
     - If CI entirely > 0 → A > B (or B > A depending on direction)
     - If CI includes 0 → A and B are incomparable
  2. Build partial order relation from all pairwise dominance
  3. Compute transitive reduction: remove edge A→C if A→B→C exists
  4. Layered layout (Sugiyama-style):
     - Assign layers by longest path from sources
     - Within-layer ordering to minimize edge crossings (barycenter heuristic)
     - Render as SVG: rectangles for treatments, downward arrows for "is worse than"
- **Rendering**: Inline SVG (no external library). Nodes colored by P-score gradient. Treatments on same layer are incomparable.
- **Location**: New section in Rankings tab after rankogram, titled "Partial Treatment Order (Hasse Diagram)".
  - Configurable significance threshold (default 0.05)
  - Option to use prediction intervals instead of confidence intervals
  - SVG export via serialization; PNG export via canvas rasterization (not Plotly — Hasse is native SVG)
- **Dependencies**: Pairwise effect estimates + CIs from NMA results.
- **Estimated lines**: ~350
- **Citation**: Rücker & Schwarzer 2017 (netmeta `hasse()` function).

### Feature 5: Contextualized GRADE Frameworks
- **What**: Extends CINeMA/GRADE with minimally and partially contextualized approaches.
- **Minimally contextualized algorithm**:
  1. For each treatment vs every other: compute effect + CI
  2. Apply MID threshold (from Feature 2) to classify effect magnitude:
     - |effect| < MID → "trivial difference"
     - MID <= |effect| < 2*MID → "small but important"
     - |effect| >= 2*MID → "large important"
  3. Cross with CINeMA certainty rating:
     - High certainty + large benefit → "among the best"
     - Low certainty → "cannot classify" regardless of effect
  4. Final category per treatment: "Among the best" / "Probably among the best" / "Intermediate" / "Probably among the worst" / "Among the worst"
- **Partially contextualized** (extends minimally):
  1. Adds direction: benefit vs harm categories
  2. Maps to 7-category scale: Large benefit / Moderate benefit / Small benefit / Trivial / Small harm / Moderate harm / Large harm
  3. Each cell shows effect category + certainty → contextual conclusion
- **Location**: New section in CINeMA tab (not GRADE tab — extends CINeMA ratings directly), titled "Contextualized Evidence Framework".
  - Two sub-tabs: "Minimally Contextualized" and "Partially Contextualized"
  - League-table-style matrix with traffic-light coloring
  - Requires MID threshold input (shared with Feature 2)
  - Auto-generates plain-language summary per comparison
- **Dependencies**: CINeMA ratings (existing), MID threshold (Feature 2), pairwise effects + CIs.
- **Estimated lines**: ~300
- **Citation**: Brignardello-Petersen et al. 2020; Nikolakopoulou et al. 2021.

## Batch 3: New Analysis Paradigms (~1,400 lines)

### Feature 6: Class Effect Models
- **What**: Hierarchical models where treatments belong to drug classes. Effects partially pooled within class.
- **Algorithm**:
  Two model types, both frequentist:

  **Model A — Common class effect** (all treatments in a class share identical effect):
  1. User assigns treatments to classes via dropdown (new "Class" column in data table)
  2. Construct reduced design matrix X_class: replace treatment indicators with class indicators
     - If class C contains treatments {t1, t2, t3}, all three map to column "C"
     - This is a constrained NMA where d_t1 = d_t2 = d_t3 = d_C
  3. Fit GLS: `beta_class = (X_class' Sigma^{-1} X_class)^{-1} X_class' Sigma^{-1} y`
  4. Standard errors from `(X_class' Sigma^{-1} X_class)^{-1}`
  5. Model fit: Q_class statistic; compare Q_class vs Q_unconstrained via chi-squared test (df = T - K where T = treatments, K = classes)

  **Model B — Exchangeable class effect** (meta-regression approach):
  1. Run unconstrained NMA first → get treatment effects d_t vs reference
  2. Meta-regression: `d_t = alpha_class(t) + epsilon_t` where alpha_class is class mean
     - This is network meta-regression with class as categorical covariate
     - Reuses existing `NetworkMetaRegression` infrastructure with class indicator as covariate
  3. Within-class heterogeneity: tau^2_within = residual heterogeneity from the meta-regression
  4. Between-class heterogeneity: variance of alpha_class estimates
  5. Treatment-level shrunk estimates: `d_t_shrunk = w * d_t + (1-w) * alpha_class` where w = tau^2_within / (tau^2_within + se_t^2)

  Note: Full Bayesian exchangeable class models (as in multinma) require MCMC. The frequentist meta-regression approach is an approximation but avoids MCMC complexity.

  **Outputs** (both models):
  - Class-level league table (class vs class effects + CIs)
  - Treatment-level league table (with class-adjusted estimates for Model B)
  - Class-level forest plot
  - Within-class heterogeneity statistics per class
  - Model comparison: AIC of class model vs unconstrained NMA
  - Goodness-of-fit test (chi-squared for common class constraint)
- **Location**:
  - **Data tab**: New "Class" column in study data table (dropdown per treatment, auto-populated with unique class names)
  - **Analysis settings**: Toggle "Enable Class Effects" + model type selector (Common / Exchangeable)
  - **Results**: New "Class Effects" sub-section in Forest Plot area + separate class-level league table
- **Dependencies**: Existing NMA engine for unconstrained estimates. Existing `NetworkMetaRegression` for exchangeable model. New reduced design matrix construction for common model.
- **Estimated lines**: ~450
- **Citation**: Phillippo et al. 2025 (multinma class effects guide); Lu & Ades 2006.

### Feature 7: Disconnected Network CNMA
- **What**: Component NMA that bridges disconnected sub-networks via shared components.
- **Algorithm** (extending existing `EnhancedCNMA` module at ~line 2392, which is the one called from UI at ~line 7190):
  1. Run connectivity check (existing `NetworkGuardian`). If connected → standard CNMA.
  2. If disconnected:
     - Decompose each treatment into components (existing CNMA parser)
     - Check if component set bridges all sub-networks (components must appear in >1 sub-network)
     - If bridgeable: fit additive component model across full network
       - Design matrix maps studies to component contrasts
       - Weights from study-level variances
       - Solve via weighted least squares
     - If NOT bridgeable (no shared components): report error with explanation
  3. Output: component effects, reconstructed treatment effects, model fit statistics
  4. Prominent warning: "Disconnected network CNMA relies on additivity assumption which cannot be verified from the data. Results should be interpreted with caution."
- **Location**: Extends existing `EnhancedCNMA` module. Auto-detects disconnected networks. Warning banner when assumptions are invoked.
- **Dependencies**: Existing `EnhancedCNMA` + `NetworkGuardian` modules.
- **Estimated lines**: ~350
- **Citation**: Rücker et al. 2020 (`discomb` in netmeta).

### Feature 8: Survival NMA Tab
- **What**: Dedicated tab for time-to-event network meta-analysis.
- **Primary mode (HR-based)**:
  1. New data table: Study, Treatment1, Treatment2, HR, LowerCI, UpperCI (auto-converts to logHR + SE)
     - Also accepts: logHR + SE directly
     - Also accepts: Events1, N1, Events2, N2 (computes logOR as approximation with warning)
  2. Runs standard NMA on log-HR scale using existing FrequentistNMA/BayesianNMA engine
  3. All existing NMA outputs apply: forest plot (HR scale), league table (HR), rankings, consistency, etc.
  4. Forest plot x-axis labeled "Hazard Ratio" with log-scale tick marks
- **Parametric extension (optional)**:
  1. Additional inputs per arm: distribution type (Weibull/Gompertz/log-normal/log-logistic/gen-gamma), shape, scale
     - Helper: "Convert from median survival" button (given median + distribution → shape/scale)
  2. Survival curve S(t) computation:
     - Weibull: S(t) = exp(-(t/scale)^shape)
     - Gompertz: S(t) = exp(-(scale/shape)*(exp(shape*t) - 1))  [shape can be negative (deceleration); scale must be > 0]
     - Log-normal: S(t) = 1 - Phi((log(t) - mu) / sigma)
     - Log-logistic: S(t) = 1 / (1 + (t/scale)^shape)
     - Gen-gamma: via numerical integration
  3. Survival curve overlay plot (Plotly): one S(t) curve per treatment, time horizon configurable
  4. RMST (Restricted Mean Survival Time): numerical integration of S(t) from 0 to t*
     - RMST difference between treatments + CI via delta method
  5. Hazard function h(t) overlay plot
  6. Median survival time comparison table
- **Location**: New "Survival NMA" tab in main tab navigation bar (after existing tabs).
  - Tab contains: data entry panel, HR-based analysis section, parametric curves section
  - Shares the same NMA engine — survival tab sets effectMeasure = "HR" internally
  - Note: HR already exists in the main effect measure dropdown (line 36). The Survival tab provides a dedicated HR-focused data entry workflow (HR/CI or logHR/SE inputs) distinct from the main tab's event-count-based approach. The main tab's HR option computes from 2x2 tables (approximate); the Survival tab accepts published HRs directly.
- **Dependencies**: Existing NMA engine (reused). New parametric survival math. Plotly for curves.
- **Estimated lines**: ~600
- **Citation**: Woods et al. 2010 (fractional polynomial NMA); Dias et al. 2018 (survival NMA framework).

## Cross-Cutting Concerns

### Shared MID Threshold
Features 2, 4, and 5 all use MID. Implement a single `AppState.midThreshold` value with:
- Global setting in Analysis Settings area
- Default by effect measure: logOR=0.5, logRR=0.2, SMD=0.2, RD=0.05, logHR=0.2
- All three features read from same source

### Error Handling
- All new modules wrapped in try-catch with user-visible error messages
- Graceful degradation: if a feature fails, it shows error card but doesn't break other analyses
- Contribution matrix and class effects only available for frequentist models (show info message for Bayesian)

### Performance
- POTH: O(T*T) — trivial for typical NMA (T<30)
- MID rankings: O(nSim * T^2) — reuses existing simulation, adds tied-rank pass
- Contribution matrix: O(T^3) — matrix inversion already done in NMA
- Hasse: O(T^2) for pairwise comparison + O(T^3) for transitive reduction
- Class effects: O(T*k) where k = number of classes
- Survival curves: O(T * nPoints) for curve computation — negligible
- All well within browser capability for typical networks (T < 50 treatments)

### Testing Strategy
- Each feature gets a validation check in the existing test runner
- POTH: verify against hand-computed example (3 treatments, known rank probs)
- MID rankings: verify ties are correctly handled; edge case MID=0 reproduces original rankings
- Contribution matrix: verify row sums = 100%; verify against netmeta::netcontrib for demo dataset
- Hasse: verify transitive reduction is correct; verify incomparable pairs
- Contextualized GRADE: verify categories match published examples
- Class effects: verify class-level pooling matches manual weighted average
- Disconnected CNMA: verify component effects; verify error on non-bridgeable networks
- Survival NMA: verify HR conversion; verify Weibull S(t) against known values

### Accessibility
- All new sections have appropriate ARIA labels
- Hasse diagram SVG includes role="img" and aria-label with text description
- New input fields have associated labels
- Color-coding always paired with text labels (never color-only information)
